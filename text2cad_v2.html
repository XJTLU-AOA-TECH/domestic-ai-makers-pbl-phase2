02

什么是 Text-to-CAD？

💬

自然语言输入

用户用日常语言描述设计需求，如"创建一个带中心孔的矩形块，四角倒圆角"

🧠

AI 理解意图

AI 提取文本中的几何特征、尺寸约束、拓扑关系，转化为机器可理解的描述

📐

3D 模型输出

生成可执行的 CAD 文件或三角网格，用于制造、游戏、仿真等不同场景

🎯 三种完全不同的技术路线

Text2CAD（Sadil Khan）：训练一个深度学习模型，让它像 GPT 写小说一样，一个字一个字地"写出"CAD 建模命令序列。优点是参数化可编辑，缺点是只支持简单形状。

earthtojake CAD Skills：不给 AI 训练模型，而是给 Claude/Codex 等编程助手一套"技能说明书"，让它直接写 Python 代码来生成 CAD。优点是工程化完备，缺点是需要调用外部 LLM。

混元3D（Hunyuan3D）：训练一个超大规模扩散模型，像 Stable Diffusion 生成图片一样，从一团"噪声"中逐步"生长"出精细的三角网格。优点是视觉质量极高，缺点是不可参数化编辑。

Text2CAD 原理通俗解读

🔤 核心思想：把 CAD 建模当成"写作文"

Text2CAD 将 CAD 建模过程看作一个序列：先画草图 → 再拉伸 → 再倒角 → 再布尔运算……就像写作文是一个字一个字接下去写。模型训练的目标就是：给定一段文字描述，让 AI "接龙"写出完整的 CAD 操作序列。

1
文本输入
"画一个长方形，中间挖个圆孔"
→
2
BERT 理解
把文字变成数学向量
→
3
Transformer 接龙
逐个预测 CAD 命令
→
4
执行生成
输出可编辑 STEP
📊

数据如何来的？

用 Mistral 大语言模型给 17 万个 CAD 模型自动写文字描述，再用 LLaVA 视觉模型检查描述和渲染图是否一致。生成 4 个难度级别（从"两个同心圆柱"到精确坐标）。

⚙️

量化是什么？

CAD 里的坐标、距离、角度都是小数。Text2CAD 把它们"四舍五入"到 256 个档位（8bit），就像把连续的颜色变成有限的调色板，让模型能当作"单词"来预测。

05

Text2CAD 公式详解

公式 ①：自回归生成（像 GPT 一样接龙）
P(C | T) = P(c1 | T) × P(c2 | c1, T) × ... × P(cN | c1...cN-1, T)
💡 通俗解释

这个公式的意思是：生成整个 CAD 序列的概率，等于"在知道文本 T 的前提下，先猜第 1 个命令是什么" × "在知道文本和第 1 个命令的前提下，猜第 2 个命令是什么"……以此类推。就像 GPT 生成文章：每写下一个字，都要看前面已经写了什么。

公式 ②：BERT 文本编码（把文字翻译成数学语言）
Tvec = AdaptiveLayer( BERT(T) )
💡 通俗解释

BERT 是一个预训练的语言理解模型，它把"画一个长方形"这句话变成一串 1024 个数字的向量。Adaptive Layer 是一个"翻译官"，把通用的语言向量转换成 CAD 模型能理解的格式。

公式 ③：训练损失（猜错了要扣分的游戏规则）
L = − log Pmodel(ctrue | context)
💡 通俗解释

Cross-Entropy 损失就像考试打分：模型预测下一个命令时，会给 256 个可能的命令各自打一个概率分。如果真实答案是"画圆"，模型给"画圆"的概率越高，扣分越少；如果猜错了，扣很多分。训练就是让模型少扣分。

earthtojake 通俗解读

🔧 核心思想：不给 AI 训练模型，而是教它写代码

earthtojake/text-to-cad 不是深度学习模型，而是一套给 Claude Code / Codex 等 AI 编程助手使用的"技能说明书"（Skills）。它的思路很简单：既然 LLM 已经会写 Python 代码了，那我们就教它如何用 build123d 等 CAD 库写出生成 3D 模型的代码，然后直接运行代码得到 CAD 文件。

1
用户说需求
"帮我做一个电机支架"
→
2
Claude 读 Skill
加载 CAD 技能说明书
→
3
Claude 写 Python
调用 build123d 库
→
4
执行出 CAD
输出 STEP / STL 等
📐

CAD Skill

生成 STEP 零件/装配体，导出 STL、3MF、GLB。支持从 Python 源文件重新生成。

🔍

CAD Viewer

本地浏览器预览 CAD、G-code 和 URDF 机器人文件，无需上传到云端。

🔩

step.parts

查找螺丝、轴承、电机等现成标准件的 STEP 文件，直接下载使用。

🖨️

G-code / Bambu

用真实切片软件（如 PrusaSlicer）将网格切片为 G-code，甚至直接发送到拓竹打印机。

🤖

URDF / SDF

生成机器人描述文件（URDF）和仿真世界文件（SDF），包含关节、约束、传感器。

📄

DXF / SendCutSend

生成 2D 激光切割图（DXF），并自动检查是否满足 SendCutSend 上传要求。

earthtojake 路线深度解析

🧠 为什么走 LLM 写代码路线？

传统 Text-to-CAD 需要收集海量数据、训练昂贵模型、只支持简单形状。而 earthtojake 的思路是：LLM 已经理解了几何、数学、物理和工程常识，不需要重新训练。只要给它正确的工具（build123d Python 库）和清晰的说明书（Skill 文件），它就能直接写出生成复杂 CAD 的代码。

✅ 优势

  • 零训练成本：不需要 GPU 集群，利用现成 LLM
  • 无限复杂度：只要 Python 能表达，就能生成
  • 完全可编辑：修改 Python 参数即可改模型
  • 工程闭环：从设计到切片到打印一条龙
  • 本地运行：100% 开源，不依赖云服务
  • 制造就绪：直接输出 STEP / G-code

⚠️ 局限

  • 依赖 LLM 能力：Claude/Codex 质量决定输出质量
  • 需要编程环境：必须安装 Python + build123d
  • 迭代式交互：通常需要多轮对话修正
  • 无视觉验证：需要用户或 Snapshot 检查几何
  • 创意受限：受限于 Python 库的建模能力
# 本质：没有数学公式，只有一个"提示工程"流程
# Prompt = SKILL.md + user_request + cad_library_docs
# Output = LLM(Codex/Claude) writes Python → executes → CAD
# 核心创新：把 CAD 生成从"训练神经网络"变成"写 prompt 工程"

混元3D 原理通俗解读

🎨 核心思想：从"一团雾"中慢慢凝聚出 3D 形状

混元3D 采用两阶段生成：先由 Hunyuan3D-DiT 生成几何形状（就像雕塑家从石头里凿出轮廓），再由 Hunyuan3D-Paint 给模型"上色"（画纹理）。形状生成使用 Flow Matching 技术——可以理解为：从一团随机的高斯噪声开始，沿着一条"直线"逐步走向真实数据，模型学习的是"每一步往哪走"。

1
输入图像
一张产品照片
→
2
DINOv2 编码
提取图像特征
→
3
DiT 去噪
从噪声生成形状
→
4
VAE 解码
恢复为三角网格
→
5
Paint 上色
生成 PBR 纹理
🏗️

DiT 是什么？

Diffusion Transformer。传统扩散模型用 U-Net，混元3D 改用 Transformer（和 GPT 同款架构）来处理 3D 数据。它把 3D 形状压缩成一系列"token"，然后在潜空间里做注意力计算。

⚡

FlashVDM 加速

标准版需要几十步去噪，Turbo 版只用 5~8 步。通过"渐进式流蒸馏"把大模型的知识"教"给小步数模型，实现 1 秒生成，质量保留 90%。

09

混元3D 公式详解

公式 ①：Flow Matching 直线路径（从噪声走向真实数据）
xt = (1 − t) · 噪声 + t · 真实数据,     t 从 0 到 1
💡 通俗解释

想象你在地图上从 A 点（一团随机噪声）走到 B 点（真实的 3D 模型）。最简单的路线就是直线。t=0 时你在 A 点（全是噪声），t=1 时你在 B 点（完美模型）。这个公式描述的就是你在这条直线上的位置。模型要学习的是："在任意位置，我应该往哪个方向走？"

公式 ②：速度场预测（告诉每个点"往哪走"）
速度 = 真实数据 − 噪声
损失 = ‖ 模型预测的速度 − 真实速度 ‖2
💡 通俗解释

如果直线路径从 A 到 B，那方向就是 B−A。模型要预测的就是这个方向（速度场）。训练时，我们给模型看直线上某个点，让它猜方向。如果猜对了，损失小；猜错了，损失大。这是 MSE（均方误差）——就像打靶，离靶心越远，扣分越多。

公式 ③：推理采样（沿着速度场积分）
最终模型 = 噪声 + ∫模型预测的速度 dt
💡 通俗解释

积分符号 ∫ 就像"把每一步的小步长加起来"。实际推理时，我们从纯噪声出发，模型告诉你"这一步往左走 0.3"，然后你走 0.3；下一步模型说"往右走 0.5"，你再走 0.5……重复 5~50 步，最终凝聚成一个完整的 3D 形状。

三路线详细对比

对比维度 Text2CAD earthtojake 混元3D
技术路线 训练 Transformer 模型 LLM 写 Python 代码 训练扩散大模型
核心方法 自回归生成离散 token 序列 Prompt 工程 + build123d 代码 Flow Matching + DiT

训练成本 高（1×A100 训练 22 天） 无（零训练，用现成 LLM） 极高（大规模 3D 数据）

输出格式 参数化 CAD（可编辑历史） STEP / STL / DXF / URDF GLB / OBJ + PBR 纹理

局限性与未来展望

⚠️

Text2CAD 局限

1. 数据集以矩形和圆柱为主，复杂曲面支持不足
2. 仅支持 sketch + extrude，缺少 revolve、sweep
3. 序列长度受限（272 tokens），无法生成复杂装配体
4. 缺乏标准化 benchmark，跨论文对比困难

⚠️

earthtojake 局限

1. 依赖 Claude/Codex 质量，不同 LLM 表现差异大
2. 需要本地 Python + build123d 环境配置
3. 通常需要多轮对话迭代才能生成正确几何
4. 受限于 Python CAD 库的表达能力

⚠️

混元3D 局限

1. 输出为"死"网格，无法参数化编辑尺寸
2. 无法直接输出 STEP 用于 CNC 精密加工
3. 需要高算力 GPU，端侧 mini 版质量下降
4. 复杂内部结构（如螺纹孔）生成不稳定

🔄

未来融合方向

路线融合：用混元3D 生成视觉概念 → 用 LLM 反推参数化 CAD 代码
Loop Engineering：设计→生成→检查→修改→重新生成的闭环
制造一体化：Text-to-CAD 直接对接 CAM，输出 G-code
教育普及：降低 CAD 学习门槛，让中学生也能设计

12

一句话总结

📐 Text2CAD = 教 AI 背 CAD 语法书

像训练 GPT 写作文一样，用 17 万个例子教 AI 背下 CAD 建模的"语法规则"。优点是规则严谨、输出可编辑；缺点是只会背过的规则，遇到复杂形状就"卡壳"。

🔧 earthtojake = 给 AI 配一套 CAD 工具箱

不给 AI 上课，而是给它一本"说明书"和一套 Python 工具。AI 会自己推理怎么组合工具。优点是不需要训练、支持无限复杂工程流程；缺点是对 AI 智商要求高，偶尔"手抖"写错代码。

🎨 混元3D = 让 AI 当 3D 雕塑家

像 Stable Diffusion 画图片一样，从一团噪声中"雕"出精细网格。优点是视觉惊艳、细节丰富；缺点是雕出来的模型像石头——好看但无法修改参数，也不能直接上机床。
