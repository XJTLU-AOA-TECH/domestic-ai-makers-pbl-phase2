模型有了，为什么还不能直接打印？

上午 AI 生成了漂亮的 3D 模型，但 3D 打印机看不懂它。

🧠 3D 打印机只懂一种语言

你手里的 STL 文件（三角网格）描述的是"这个零件长什么样"。但 3D 打印机不关心"样子"，它只关心一件事：

"喷头现在走到哪？挤多少料？温度多少？"

这种"一步一步告诉机器怎么做"的指令，就是 G-code。

🔪 切片 = 翻译官

切片软件（Slicer）的工作，就是把 3D 模型翻译成 G-code。类比一下：

场景输入翻译工具输出
编程C++ 代码编译器机器码（0101）
3D 打印STL 模型切片软件G-code（喷头指令）

切片软件要决定的事情非常多——不只是"翻译"，还要做大量制造决策。

🤔 切片软件要决定什么？

STL 模型 → 层厚多少？ → 内部填充多少？ → 悬空要不要支撑？ → 外壳打几层？ → G-code

每一个决策都影响最终零件的强度、精度、时间和耗材。

G-code：3D 打印机的"机器语言"

打开 G-code 文件，你会看到打印机执行的每一行指令。

📄 G-code 长什么样？

; 这是注释，打印机不会执行
G28          ; 归零——喷头回到原点
M104 S200    ; 设置喷头温度 200°C
M140 S60     ; 设置热床温度 60°C
G1 X50 Y30   ; 喷头移动到坐标 (50, 30)
G1 X60 Y30 E1.5 F1500
             ; 移动到 (60,30)，挤出 1.5mm 料，速度 1500mm/min
G1 Z0.2      ; 喷头上升到 0.2mm（第一层高度）

每一层（比如 0.2mm 厚）都有几百到几千行这样的指令。一个普通零件的 G-code 文件可能有几万行。

🔑 关键指令速查

指令含义类比
G1直线移动（同时可以挤出料）"走到那，边走边画"
G28归零（回原点）"回到起点"
G1 Z0.2Z 轴上升到 0.2mm"换一层"
M104 S200喷头加热到 200°C"预热笔"
M140 S60热床加热到 60°C"预热纸"
E1.5挤出 1.5mm 长的料"挤出颜料"
F1500移动速度 1500mm/min"走快点"

切片软件就是自动生成这些指令的"程序员"。你不需要手写 G-code，但理解它有助于排查打印问题。

切片参数：每个选项都在决定什么？

切片软件右侧有一堆参数，它们不是随便填的——每个都影响最终零件。

📏 层厚（Layer Height）

每一层塑料的厚度。

推荐：0.2mm

0.1mm = 精细但慢（2 倍时间）
0.3mm = 快速但粗糙

🧱 壁厚（Wall Thickness）

外壳打几层。 nozzle 0.4mm 宽，3 层 = 1.2mm 厚。

推荐：3 层

太薄 = 一捏就瘪
太厚 = 浪费料、时间翻倍

🐝 填充密度（Infill）

内部不是实心的，而是蜂窝/网格结构。

推荐：20%

0% = 完全空心（花瓶）
100% = 实心（重、慢、贵）
20% 是强度与重量的最佳平衡

🌡️ 温度（Temperature）

喷头熔化塑料的温度。

PLA：200°C / 热床 60°C

太低 = 料挤不出来
太高 = 拉丝、变形

🦶 支撑（Support）

悬空部分（>45°）需要支架托住，否则塑料会塌。

推荐：自动（45°阈值）

支撑打完后要手动拆掉，会在表面留痕迹

⚡ 速度（Speed）

喷头移动速度。

推荐：默认即可

太快 = 精度下降、层间粘合差
太慢 = 时间太长

🎯 一句话记忆

层厚决定表面光不光滑，壁厚决定外壳硬不硬，填充决定内部实不实，支撑决定悬空掉不掉，温度决定料粘不粘得牢。

完整流程：从 STL 到 G-code

以下是用 Bambu Studio 处理 AI 生成零件的标准流程。

Step 1：导入模型

文件 → 导入 → 选择 .stl

↓
模型出现在 3D 视图中，可能方向不对
Step 2：摆放与旋转

最大平面朝下，减少支撑

↓
选中模型 → 点击旋转工具 → 调整角度
Step 3：检查与修复

点击"修复"按钮，检查破面/非流形

↓
如果 AI 生成的模型有缺陷，这里会标红。需要回炉修正 Prompt。
Step 4：设置参数

层厚 0.2mm / 填充 20% / 壁厚 3 层 / 支撑自动

↓
右侧参数面板，逐项确认
Step 5：切片

点击"切片"按钮 或 Ctrl+G

↓
软件计算每一层路径，显示预计时间和耗材
Step 6：预览检查

逐层拖动滑块，观察支撑、壁厚、孔是否完整

↓
重点看：悬空有没有支撑？孔有没有被填满？壁厚均匀吗？
Step 7：导出 G-code

导出 → 保存 .gcode 文件

↓
可以发送到打印机，或保存到 SD 卡/U 盘
✅ 流程完成

你现在有一份 .gcode 文件，可以直接交给 3D 打印机执行。从 AI 生成自然语言描述，到拿到可打印的指令，完整链路走通了。

制造约束：设计时就要考虑

切片时发现的问题，根源往往在设计阶段。这是"制造约束前置"的理念。

⚠️ 6 条铁律

约束为什么AI Prompt 怎么写
最小壁厚 ≥ 1.2mm nozzle 0.4mm，3 层才能成墙 "wall thickness 3 mm"
悬空角度 ≤ 45° 超过 45° 塑料会塌，需要支撑 "avoid overhangs above 45 degrees"
孔径 ≥ 2mm 太小打不圆，变成多边形 "hole diameter 8 mm"（不要小于 2）
层高是特征高度的整数倍 否则特征高度不准 特征高度 0.4/0.6/0.8mm，配合 0.2mm 层厚
支撑不能封死内部 拆不出来，零件报废 "internal cavities accessible"
最大平面朝下 接触热床面积大，粘得牢 切片时手动旋转，不是 Prompt 的事
💡 关键洞察

如果切片时发现某个零件"打不了"，不要只怪切片软件——回到上午的 Prompt，看看是不是设计时就没考虑这些约束。Loop Engineering 的验收标准里，应该加入制造约束检查。

常见问题速查

现象原因解决
导入 STL 报错 模型有破面/非流形 Bambu Studio 修复，或回炉重生成
切片后孔被填满 孔径太小或 STL 缺陷 Prompt 孔径 ≥ 2mm，重新导出 STL
悬空没生成支撑 支撑角度阈值设太高 从 45° 调到 50° 或手动加支撑
预计时间太长 填充太高或层厚太薄 填充改 15%，层厚改 0.28mm
薄壁打不圆 壁厚不是 nozzle 整数倍 壁厚设为 0.4/0.8/1.2/1.6mm
支撑拆不下来 支撑封死了内部空腔 切片预览检查支撑位置，或调整模型
