3D 建模：从网格到参数化

在让 AI 帮你建模之前，先搞清楚"3D 文件"到底有几种，为什么工程师偏爱 STEP。

🧊 两种 3D 文件的本质区别

特性STL / OBJ（三角网格）STEP（参数化 CAD）
本质一堆三角形拼成的"壳"带几何参数的工程描述
尺寸固定，无法精确修改可编辑，改一个数字全更新
圆孔多边形近似，不是真圆数学意义上的精确圆
壁厚不知道，只有表面明确记录，可测量
用途3D 打印、游戏、渲染机械设计、数控加工、装配
比喻一张照片（能看不能改）一份图纸（能改能生产）

结论：STL 是"给消费者看的"，STEP 是"给工程师用的"。Text-to-CAD 输出 STEP，意味着 AI 生成的零件可以直接进入工厂生产流程。

🎨 传统 3D 建模是怎么做的？

工程师打开 SolidWorks / CATIA / Fusion 360，按以下流程：

画草图 → 拉伸/旋转 → 打孔/倒角 → 装配 → 导出 STEP

一个简单零件需要 30 分钟，复杂零件需要几天。而且每一步都需要人脑理解设计意图。

Text-to-CAD：AI 翻译官

你说人话，AI 出图纸。不是出图片，是出真正的工程文件。

🤖 什么是 Text-to-CAD？

Text-to-CAD 是 AI 辅助设计的新范式。你把"我要一个带四个螺丝孔的矩形块"这类自然语言丢给它，它输出的是 .step 文件——不是渲染图，不是网格模型，而是工程师能直接拿去 CNC 加工、3D 打印、装配验证的参数化 CAD 模型。

自然语言 → Text-to-CAD → STEP 文件 → 制造/装配

📐 为什么 Prompt 要像工程图纸语言？

AI 听懂了"意思"，但不一定精确执行"尺寸"。比如你说"做一个大一点的孔"，AI 不知道"大一点"是 1mm 还是 10mm。

❌ 模糊的 Prompt✅ 精确的 Prompt
"做一个方块""Create a 100 x 60 x 20 mm block"
"打几个孔""four 8 mm vertical through-holes at X=±35, Y=±20"
"倒个角""2 mm chamfer on the top outer perimeter only"

原则：带单位（mm）、带数量、带位置、带约束。越像工程图纸，AI 越准。

🔗 项目来源

本课程基于 earthtojake/text-to-cad 开源项目。这是一个 Skills 库——不是独立软件，而是给 Claude Code / Codex 等 AI Agent 用的 CAD 能力插件。

Skills：给 AI 装"扳手"

Claude Code 本身不会画 CAD。Skills 是它的"外挂工具箱"。

🧰 什么是 Skills？

Skills 是 Anthropic 提出的Agent 工具插件系统。你可以把它理解为：Claude Code 是一个聪明的工人，但它出厂时只有一双手。Skills 就是给它配的扳手、螺丝刀、卡尺——让它能调用外部专业工具。

Text-to-CAD 就是其中一个 Skill。安装后，Claude Code 就能：

  • 把自然语言翻译成 CAD 代码
  • 生成 STEP / STL / DXF 等工程文件
  • 检索标准件（螺丝、轴承、电机）
  • 生成机器人 URDF 描述文件

发现：AI 不是完美的

先动手生成一个零件，你会发现问题——这就是 Loop Engineering 的出发点。

🔬 先做个实验

运行以下命令，生成一个"20mm 立方体"：

npx skills run cad --prompt "Create a 20 mm cube" --output ./cube.step

然后用 FreeCAD 打开测量。你大概率会发现：

  • 边长不是精确的 20.00mm，可能是 19.85 或 20.12
  • 如果带孔，孔径可能偏差 0.5mm 以上
  • 复杂特征（倒角、加强筋）可能缺失或位置不对

问题出现了：AI 听懂了你的意思，但执行有误差。对于工程应用来说，1mm 的误差可能就是废品。

😤 手动修正的循环

传统做法是：你测量 → 写修正指令 → 再生成 → 再测量……

# 第一轮

你: "生成一个 50mm 立方体，中心 20mm 通孔"

AI: 生成零件（孔径 19.8mm）

# 你打开 FreeCAD 测量，发现孔小了

# 第二轮

你: "孔应该是 20mm 直径，不是 19.8，请修正"

AI: 生成零件（孔径 20.2mm）

# 还是不准，再改……

你变成了 "发球机"——不断给 AI 喂指令，AI 被动执行。批量生成 10 个零件时，这非常痛苦。

Loop Engineering：让 AI 自己循环改到合格

2026 年 AI 工程化的新范式——你设计循环，AI 当运动员。

🔄 三层演进

层级你做什么AI 做什么
Prompt Engineering写一句好指令给一次答案
Harness Engineering给 Agent 工具和环境在约束内自主调用工具
Loop Engineering设计"会自己下指令的系统"自主循环：行动→观察→推理→重来，直到达标

本质：你不再是 AI 的"发球机"，而是设计一个 while 循环——AI 自己判断零件是否合格，不合格就自动改，改到对为止。

🛠️ Text-to-CAD 的 Loop 长什么样？

# 你写的验收标准

target = {"边长": 50, "孔径": 20, "误差": "≤2%"}

# AI 自己循环

for i in range(5): # 最多 5 次

step = generate_cad(prompt) # 生成

actual = measure(step) # 测量

error = compare(target, actual) # 对比

if error <= 2%:

break # 合格，停

else:

prompt = f"修正：偏差 {error}%，请调整"

# 继续下一轮

你只需做一件事：设定验收标准（如"尺寸误差 ≤ 2%"）。剩下的，AI 自己搞定。

🧩 Loop 的六原语

设计一个 CAD Loop，需要这六个组件：

原语作用CAD 场景示例
指令Loop 要达成的目标"生成 50mm 立方体，误差 ≤ 2%"
工具模型能调用的能力text-to-cad、FreeCAD 测量、尺寸对比
上下文每轮注入的信息上一轮 STEP、测量结果、修正历史
记忆跨轮次状态积累记录每轮目标/实际尺寸、修正指令
终止什么时候停误差 ≤ 2% 或迭代达 5 次上限
评估结果校验和反馈尺寸对比脚本输出误差百分比

BENCHMARK 04 · LOOP 阶梯轴（带键槽）—— Loop Engineering 版

Stepped Shaft With Keyway

🔄 本零件用 Loop Engineering 生成

不再手动一发一收，而是写一个循环脚本，让 AI 自己生成→测量→修正→直到达标。

📐 零件科普

阶梯轴是传动系统的核心零件，用于安装齿轮、轴承、联轴器。关键特征：

  • 三段不同直径（20/30/20mm）
  • 中间段顶部的键槽（传递扭矩）
  • 两端倒角（便于装配）

工程意义：键槽的尺寸和位置必须精确，否则装不上齿轮。这是检验 AI 对"功能性特征"理解能力的零件。

1写 Loop 脚本

创建一个 Python 脚本 loop_shaft.py：

loop_shaft.py
import subprocess
import os

# ========== 验收标准 ==========
target = {
    "total_length": 120,      # mm
    "diameters": [20, 30, 20], # 三段直径
    "tolerance": 0.02         # 2% 误差上限
}

prompt = """Create a 120 mm shaft along X with 20/30/20 mm diameter 
stepped sections. Add end chamfers and a shallow rectangular keyway 
on top of the middle section."""

output_file = "benchmark_04_shaft.step"
max_iterations = 5

# ========== Loop 开始 ==========
for i in range(max_iterations):
    print(f"\n--- 第 {i+1} 轮 ---")

    # 1. 生成
    result = subprocess.run([
        "npx", "skills", "run", "cad",
        "--prompt", prompt,
        "--output", output_file
    ], capture_output=True, text=True)

    if result.returncode != 0:
        print("生成失败:", result.stderr)
        continue

    print(f"✅ 已生成 {output_file}")

    # 2. 测量（这里用 FreeCAD 脚本或手动测量）
    # 实际项目中，这里应该调用 FreeCAD 的 Python API 自动测量
    print("📏 请用 FreeCAD 测量以下尺寸并输入：")
    actual_length = float(input("  总长度 (mm): "))
    actual_d1 = float(input("  第一段直径 (mm): "))
    actual_d2 = float(input("  第二段直径 (mm): "))
    actual_d3 = float(input("  第三段直径 (mm): "))

    # 3. 计算误差
    errors = [
        abs(actual_length - target["total_length"]) / target["total_length"],
        abs(actual_d1 - target["diameters"][0]) / target["diameters"][0],
        abs(actual_d2 - target["diameters"][1]) / target["diameters"][1],
        abs(actual_d3 - target["diameters"][2]) / target["diameters"][2],
    ]
    max_error = max(errors)

    print(f"  最大误差: {max_error*100:.1f}%")

    # 4. 判断是否合格
    if max_error <= target["tolerance"]:
        print(f"🎉 合格！共迭代 {i+1} 轮")
        break
    else:
        # 5. 自动修正 Prompt
        prompt += f"\n修正：当前最大误差 {max_error*100:.1f}%，请精确调整尺寸。"
        print(f"🔄 误差超标，自动修正 Prompt，进入下一轮...")
else:
    print(f"⚠️ 达到最大迭代次数 {max_iterations}，请检查 Prompt 或手动调整。")

print(f"\n📄 最终文件: {output_file}")
print(f"📊 迭代报告: 共 {i+1} 轮，最终误差 {max_error*100:.1f}%")
💡 说明

这个脚本演示了 Loop 的核心逻辑。实际工程中，测量步骤可以用 FreeCAD 的 Python API（import Part）自动完成，不需要手动输入。

2运行 Loop
python loop_shaft.py

按提示输入 FreeCAD 测量结果，观察 AI 如何自动修正。

  • loop_shaft.py 脚本可运行
  • 记录迭代轮数和最终误差
BENCHMARK 05 · LOOP 电子外壳（带支撑柱）—— Loop Engineering 进阶

Open-Top Electronics Enclosure With Bosses

🔄 本零件用 Loop Engineering 生成，加入结构检查

除了尺寸，还要检查"结构完整性"——壁厚、支撑柱、盲孔是否都存在。

📐 零件科普

电子外壳是 3D 打印最常见的应用场景。关键特征：

  • 空心结构（壁厚 3mm，省材料）
  • 内部支撑柱（standoff，用于固定电路板）
  • 支撑柱上的盲孔（不通到底，用于拧螺丝）
  • 外角圆角（2mm，避免打印时应力开裂）

工程意义：这是第一个"薄壁+内部特征"的零件，考验 AI 对制造约束（壁厚、打印方向）的理解。

1写进阶 Loop 脚本

创建 loop_enclosure.py，加入结构检查：

loop_enclosure.py
import subprocess

# ========== 验收标准（尺寸 + 结构） ==========
target = {
    "wall_thickness": 3,      # mm
    "corner_fillet": 2,       # mm
    "standoff_count": 4,      # 支撑柱数量
    "tolerance": 0.02
}

prompt = """Create a hollow open-top enclosure with 3 mm walls and floor. 
Add four internal standoffs with centered blind holes and 2 mm outside 
vertical corner fillets."""

output_file = "benchmark_05_enclosure.step"
max_iterations = 5

for i in range(max_iterations):
    print(f"\n--- 第 {i+1} 轮 ---")

    # 1. 生成
    subprocess.run([
        "npx", "skills", "run", "cad",
        "--prompt", prompt,
        "--output", output_file
    ])

    print("📏 请用 FreeCAD 检查并输入：")

    # 2. 尺寸检查
    wall = float(input("  壁厚 (mm): "))
    fillet = float(input("  外角圆角 (mm): "))

    # 3. 结构检查（计数）
    standoffs = int(input("  支撑柱数量: "))
    has_blind_holes = input("  支撑柱是否有盲孔? (y/n): ").lower() == 'y'
    is_hollow = input("  是否为空心开口? (y/n): ").lower() == 'y'

    # 4. 评估
    errors = []
    if abs(wall - target["wall_thickness"]) / target["wall_thickness"] > target["tolerance"]:
        errors.append(f"壁厚偏差: {wall}mm")
    if abs(fillet - target["corner_fillet"]) / target["corner_fillet"] > target["tolerance"]:
        errors.append(f"圆角偏差: {fillet}mm")
    if standoffs != target["standoff_count"]:
        errors.append(f"支撑柱数量不对: {standoffs}个")
    if not has_blind_holes:
        errors.append("缺少盲孔")
    if not is_hollow:
        errors.append("不是空心结构")

    if not errors:
        print(f"🎉 结构完整 + 尺寸合格！共 {i+1} 轮")
        break
    else:
        print("🔄 发现问题:")
        for e in errors:
            print(f"   - {e}")
        prompt += "\n修正：" + "；".join(errors) + "，请修正。"
        print("自动更新 Prompt，进入下一轮...")
else:
    print(f"⚠️ 达到最大迭代次数")

print(f"\n📄 最终文件: {output_file}")
print(f"📊 迭代轮数: {i+1}")
2运行并对比体验
python loop_enclosure.py

对比零件 1-3 的手动流程和零件 4-5 的 Loop 流程，体会Loop Engineering的价值：

  • 你只需设定标准，不需要每轮都写 Prompt
  • AI 自己发现问题、自己修正
  • 批量生成时，脚本可以无人值守运行
