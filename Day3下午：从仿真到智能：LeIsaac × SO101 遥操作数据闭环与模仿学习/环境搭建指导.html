🐍 环境搭建小白书

给 LeRobot 实验同学的「手边指南」—— 不装懂，只求会

📦 为什么需要 "环境"？

很多同学第一次听到 "创建虚拟环境" 时，脑子里只有一个问号：我电脑里不是已经有 Python 了吗？为什么还要再装一个？

想象一下，你租了一间单身公寓（你的电脑），里面已经有了一个厨房（系统自带的 Python）。

现在你要做川菜（项目A），需要买辣椒、花椒、豆瓣酱；同时你室友要做甜品（项目B），需要买奶油、面粉、香草精。

如果全部塞进同一个冰箱，会串味，而且可能互相污染（比如辣椒粉撒进奶油里）。

虚拟环境 = 给每个项目配一个独立的小冰箱。 川菜一个冰箱，甜品一个冰箱，互不干扰。

🤖 具体到 LeRobot

LeRobot 需要很多特定的软件包（比如 PyTorch、HuggingFace、一些机器人驱动库）。这些包的版本要求很严格：

  • PyTorch 2.1 和 2.3 可能不兼容
  • 某个库需要 numpy 1.x，另一个库需要 numpy 2.x
  • 装错了版本，代码就会报错，而且报错信息你根本看不懂
养成好习惯：每做一个新项目，先建一个新环境。 就像每学一门新课，准备一个新笔记本一样。

🏠 Anaconda vs Miniconda vs Conda

这三个词经常一起出现，很多同学搞混。简单区分：

名称 是什么 适合谁
Conda 一个「环境管理工具」本身 概念，不是软件
Anaconda Conda + 150+ 科学计算包 + 图形界面 数据科学新手，硬盘空间大
Miniconda Conda + 极少基础包，没有图形界面 推荐！ 轻量、干净、够用
做 LeRobot 实验，安装 Miniconda 就够了。Anaconda 太大太臃肿，而且那些预装的包你大概率用不上。

🛠️ 安装 Miniconda

  1. 打开官网：docs.conda.io/en/latest/miniconda.html
    认准 .edu 或 .io 结尾的官方地址，不要点百度广告
  2. 下载对应你系统的安装包：
    Windows → Miniconda3 Windows 64-bit
    Mac (Intel) → Miniconda3 MacOSX x86_64.pkg
    Mac (M1/M2/M3) → Miniconda3 MacOSX arm64.pkg
  3. 安装时 勾选 "Add Miniconda3 to my PATH"（Windows 用户特别注意）
    如果不勾选，安装完后你在命令行里输入 conda 会提示 "不是内部或外部命令"。
    如果已经忘了勾选，后面有补救方法。

⌨️ 最常用的 6 条命令

不用记全部，这 6 条够你用一年：

1. 查看现有环境

$ conda env list

带 * 星号的就是你当前所在的环境。base 是默认环境，尽量别在 base 里装东西。

2. 创建新环境（核心！）

$ conda create -n lerobot python=3.10
-n lerobot 是给环境起名字，就像给笔记本写封面。
python=3.10 是指定 Python 版本。LeRobot 推荐 3.10，不要装 3.12（太新，很多库还不支持）。

3. 激活（进入）环境

$ conda activate lerobot

激活后，命令行前面会出现 (lerobot) 字样，表示你现在在这个「小冰箱」里操作。

4. 安装包

(lerobot) $ pip install torch
在环境里优先用 pip 装包，不要用 conda install（除非官方明确要求）。pip 更轻量，版本更新更及时。

5. 退出环境

(lerobot) $ conda deactivate

6. 删除环境（重来）

$ conda remove -n lerobot --all
环境搞坏了不要慌，删掉重建是最快的解决办法。你的代码文件（在文件夹里）不会被删，删的只是那个「小冰箱」里的包。

📝 命令速查卡

你想做什么输入什么
查看所有环境conda env list

📁 路径（Path）到底是什么？

很多同学装完软件后，命令行里输入命令却提示 "不是内部或外部命令"。这个问题 90% 是因为「路径」没搞对。

想象你的电脑是一个巨大的图书馆，每个软件（比如 Python、Conda、Git）都是一本书。

「路径」就是告诉电脑：这本书放在哪个书架上。

如果你没告诉电脑书架位置，它当然找不到书，也就不知道你输入的命令是什么意思。

🗂️ 绝对路径 vs 相对路径

C:\ Users\ 你的用户名\ lerobot\ train.py

上面这一长串就是 绝对路径 —— 从「根目录」开始，完整描述文件位置。

# 绝对路径（完整地址） $ cd C:\Users\你的名字\lerobot # 相对路径（相对于你当前的位置） $ cd .. # 返回上一级文件夹 $ cd lerobot # 进入当前文件夹下的 lerobot 子文件夹
../ 是「上一级」，./ 是「当前这一级」。就像你说「上楼」和「在本层」的区别。

🔧 环境变量（PATH）怎么修

安装时如果忘了勾选 "Add to PATH"，你需要手动告诉电脑 Conda 在哪里：

Windows 补救方法

  1. 找到 Miniconda 安装位置（默认是 C:\Users\你的用户名\miniconda3）
  2. 复制这个路径，再进入里面的 Scripts 文件夹，再复制一次路径
  3. 右键「此电脑」→ 属性 → 高级系统设置 → 环境变量
  4. 找到「Path」→ 编辑 → 新建 → 把刚才复制的两个路径粘贴进去
  5. 重启命令行（必须重启！）

Mac/Linux 补救方法

# 打开配置文件 $ nano ~/.zshrc # Mac 新系统用 zsh $ nano ~/.bashrc # Linux 或旧 Mac 用 bash # 在文件末尾添加这一行（路径根据实际安装位置调整） export PATH="/Users/你的名字/miniconda3/bin:$PATH" # 保存后执行 $ source ~/.zshrc
改完环境变量后，必须重启终端（命令行窗口）才能生效。只关标签页不行，要整个关掉重开。

🎯 做实验时怎么管理文件夹

推荐的项目文件夹结构（养成好习惯，以后找东西不抓瞎）：

~/projects/lerobot-exp/
  ├── data/         # 放数据集
  ├── models/       # 放训练好的模型
  ├── scripts/       # 放自己写的脚本
  ├── train.py        # 主训练文件
  └── README.md     # 实验记录（很重要！）
永远不要在桌面或中文路径下跑代码！很多库对中文路径支持不好，会出各种奇怪错误。统一放在 C:\projects 或 ~/projects 下。

🤖 LeRobot 环境搭建：一步一步来

下面是一份「傻瓜式」操作流程，跟着做就行，不要跳步：

  1. 打开终端（命令行）
    Windows：按 Win + R，输入 cmd 回车
    Mac：按 Command + 空格，输入 Terminal
  2. 创建一个干净的环境
    $ conda create -n lerobot python=3.10 -y
    -y 表示「遇到询问直接选 yes」，省去你一直按回车的麻烦
  3. 进入环境
    $ conda activate lerobot
    看到前面出现 (lerobot) 就对了
  4. 进入你的项目文件夹
    (lerobot) $ cd ~/projects/lerobot-exp
  5. 克隆 LeRobot 仓库
    (lerobot) $ git clone https://github.com/huggingface/lerobot.git (lerobot) $ cd lerobot
  6. 安装依赖
    (lerobot) $ pip install -e . # -e . 表示「以可编辑模式安装当前文件夹里的项目」
  7. 验证安装
    (lerobot) $ python -c "import lerobot; print('OK')"
    如果输出 OK，恭喜你，环境搭好了！

💾 环境导出与分享（团队协作必备）

你配好了环境，同学电脑上没有，一个个教太麻烦。可以导出「配方清单」：

# 导出当前环境的所有包清单 (lerobot) $ pip freeze > requirements.txt # 同学拿到后，一键复刻你的环境 (lerobot) $ pip install -r requirements.txt
requirements.txt 就像奶茶店的「配方表」。你把自己的配方记下来，别人按同样的配方做，味道一模一样。

这比截图、口述、发微信靠谱一百倍。

🆘 常见错误急救室

❌ "conda 不是内部或外部命令"

原因：系统不知道 Conda 装在哪里（PATH 没配好）
解决：看第三页「环境变量怎么修」，或者卸载重装时勾选 "Add to PATH"

❌ "CommandNotFoundError: Your shell has not been properly configured"

原因：Conda 没和你的「命令行外壳」接上头
解决：
$ conda init bash # Linux/旧 Mac $ conda init zsh # 新 Mac $ conda init cmd.exe # Windows CMD
然后重启终端。

❌ "No module named 'torch'" / "No module named 'lerobot'"

原因：你在「外面」（base 环境）跑代码，但包装在「里面」（lerobot 环境）了
解决：先 conda activate lerobot，再运行代码

❌ 安装包时红字报错，满屏飞舞

原因：网络问题（连不上国外服务器）或版本冲突
解决：换国内镜像源
(lerobot) $ pip install 包名 -i https://pypi.tuna.tsinghua.edu.cn/simple
或者永久换源：
(lerobot) $ pip config set global.index-url https://pypi.tuna.tsinghua.edu.cn/simple

❌ 明明装了包，代码里 import 还是报错

排查三步走：
1. 确认你在正确的环境：conda env list 看 * 号在哪里
2. 确认包装上了：pip list | findstr 包名（Windows）或 pip list | grep 包名（Mac/Linux）
3. 确认代码文件路径没有中文或空格

🔄 终极绝招：删了重来

如果环境已经乱到你自己都搞不清装了什么，最干净的做法是：

# 1. 退出当前环境 (lerobot) $ conda deactivate # 2. 删除环境 $ conda remove -n lerobot --all # 3. 重新创建（回到第四页从头做） $ conda create -n lerobot python=3.10 -y
删环境不会删你的代码文件！代码存在文件夹里，环境只是「运行代码的厨房」。厨房脏了，换一间就行，菜（代码）还在桌上。

🎓 给同学的一句话

环境搭建是机器人学习的第一道门槛，但也是最值得花时间的投资。

你现在花 30 分钟搞懂 Conda，未来做 10 个项目、读研究生、进公司实习，这套方法完全通用。

不要怕报错，报错是电脑在教你。 复制报错信息去搜索，你会成长得飞快。

📚 Isaac Lab 装好了，做新任务还要重装吗？

完全不需要！ 这是很多同学最容易混淆的一个点。

想象你在图书馆借了一本很厚的大部头书（Isaac Lab），书已经在你桌上了。

现在你要做新任务 = 换了一本习题册。

你需要重新把大部头书借一遍吗？ 不需要。你只需要换一个新笔记本（新环境），书还是原来那本。

Isaac Lab 的文件已经躺在你的硬盘里了，不会因为你要做新任务就消失。

🎯 三种情况，分别处理

场景 要重装 Isaac Lab 吗 怎么做
新任务也是 Isaac Lab 的
比如换机器人模型、换训练算法		 ❌ 不需要 直接用现有的 isaaclab 环境
新任务也是 Isaac Lab 的，但想隔离
怕搞坏原来的环境		 ❌ 不需要 克隆环境：conda create -n isaaclab2 --clone isaaclab
新任务和 Isaac Lab 完全无关
比如 LeRobot、纯视觉项目		 ❌ 不需要 新建空白环境，不装 Isaac Lab
核心结论：三种情况都不需要重新下载 Isaac Lab。它装一次就够了。

⚠️ 为什么不建议往 Isaac Lab 环境里硬塞新包？

Isaac Lab 的环境通常很「重」，里面已经装了：

  • Isaac Sim / Omniverse（几十 GB）
  • 特定版本的 PyTorch、CUDA、Hydra
  • 各种 NVIDIA 专用库

如果你往里面再装 LeRobot 或其他项目的依赖，很容易发生：

问题 后果
版本冲突 LeRobot 要 PyTorch 2.1，Isaac Lab 绑定了 2.3，装上去 Isaac Lab 可能崩
环境臃肿 包越装越多，出问题的时候根本不知道是哪个包在捣乱
重装代价大 Isaac Lab 环境配一次可能要半小时，搞坏了重装很崩溃
Isaac Lab 环境就像实验室的「精密仪器房」——只放跟 Isaac 相关的东西，别往里面堆杂物。普通项目另开一间「普通实验室」，搞坏了重建只要 2 分钟，不心疼。

📋 最实用的技巧：克隆环境

如果你担心在 Isaac Lab 环境里装新东西会把它搞坏，可以用 conda 的「克隆」功能：

# 把原来的环境完整复制一份，改个名字 $ conda create -n isaaclab_exp2 --clone isaaclab # 然后进入新环境随便折腾 $ conda activate isaaclab_exp2
克隆是秒级的，不需要重新下载任何文件，只是复制一份清单。
你在 isaaclab_exp2 里随便折腾，搞坏了删掉就行，原来的 isaaclab 完好无损。
硬盘会多占一份空间（因为环境是独立的），但不需要重新下载 Isaac Lab 本体。

🔄 什么时候才需要重新下载 Isaac Lab？

只有以下情况才需要重新下载安装：

  • 你换了台新电脑
  • 你重装了系统
  • Isaac Lab 出了大版本更新（比如 1.x → 2.x），且你需要新功能
  • 原来的安装文件被你自己手动删掉了
记住：你之前花半小时配好的环境，就是让你反复用的，不是一次性消耗品。做新任务时，要么直接用老环境，要么 conda clone 复制一份。永远不需要「重新下载」那几十 GB 的文件。

💡 一句话总结

Isaac Lab 像一本厚书，环境像你的笔记本。

做新任务 = 换习题册，不需要重新借书。
只需要决定：用原来的笔记本，还是新开一本。
