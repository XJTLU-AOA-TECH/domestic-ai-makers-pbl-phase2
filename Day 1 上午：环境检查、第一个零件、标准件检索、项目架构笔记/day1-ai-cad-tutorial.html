Lab 01：环境检查

预计用时：15 分钟
🎯 本实验目标

确认所有必要的软件和工具已正确安装，API 可以正常连通，为后续实验做好准备。

1打开终端

打开你的终端（Windows 使用 PowerShell / CMD，Mac/Linux 使用 Terminal）。确保你已经进入项目目录。

2检查 Node.js 版本

在终端中输入以下命令，检查 Node.js 是否已安装且版本大于等于 18：

node -v
✅ 预期输出（类似）

v18.17.0 或更高版本（如 v20.x.x）

❗ 如果报错 "command not found"？

说明 Node.js 未安装或未添加到 PATH。请重新运行 setup.md 中的安装步骤。

3检查 Python 版本

继续在终端中输入以下命令：

python -V
✅ 预期输出（类似）

Python 3.10.x 或更高版本

4检查 CAD Skill 是否安装

运行以下命令查看已安装的 Skill 列表，并筛选出 cad 相关的：

npx skills list | grep cad
✅ 预期输出（应包含以下两项）

cad - CAD model generation
cad-viewer - 3D model viewer

5检查 API 连通性

运行以下 Python 命令，验证 LLM API 是否可以正常调用：

python -c "import openai; print('API OK')"
✅ 预期输出

API OK

💡 验证 API Key 的小技巧

你也可以运行 echo $OPENAI_API_KEY（Mac/Linux）或 echo %OPENAI_API_KEY%（Windows）来查看环境变量是否已设置。

Lab 01 验收标准
  • npx skills list 能显示 cad、cad-viewer 等 Skill
  • API 连通性测试通过

Lab 02：生成第一个零件

预计用时：30 分钟
🎯 本实验目标

使用 text-to-cad 工具，通过自然语言描述生成一个 20mm 立方体的 STEP 文件，并导出为 STL 格式，最后通过本地查看器查看。

1创建今日工作目录

在终端中执行以下命令，创建今天的工作目录并进入：

mkdir -p day1-ai-cad-fundamentals/lab02-first-cad
cd day1-ai-cad-fundamentals/lab02-first-cad
💡 命令解释

mkdir -p 会递归创建目录（包括父目录），如果目录已存在也不会报错。cd 是切换目录。

2生成 20mm 立方体（核心步骤）

运行以下命令，使用自然语言描述生成一个 20mm 的立方体：

npx skills run cad --prompt "Create a 20 mm cube" --output ./cube.step
ℹ️ 命令参数详解
  • npx skills run cad — 调用 cad Skill
  • --prompt "Create a 20 mm cube" — 用英文自然语言描述你想生成的零件
  • --output ./cube.step — 指定输出文件路径和格式（STEP 格式）
✅ 预期结果

命令执行后，你应该能看到进度输出，最终在 day1-ai-cad-fundamentals/lab02-first-cad/ 目录下生成 cube.step 文件。

💡 Prompt 编写技巧

务必在 prompt 中明确指定单位，如 "20 mm cube" 而不是 "20 cube"。虽然默认单位是 mm，但 LLM 可能会误解无单位的数字。

3验证 STEP 文件已生成

运行以下命令检查文件是否存在且大小大于 0：

Mac/Linux：

ls -la cube.step

Windows：

dir cube.step
✅ 预期输出（类似）

-rw-r--r-- 1 user group 4.5K Jun 23 10:00 cube.step

文件大小应大于 0（通常为几 KB 到几十 KB）。

4导出 STL 格式

将生成的 STEP 文件导出为 STL 格式（用于 3D 打印）：

npx skills run cad --input ./cube.step --export stl --output ./cube.stl
ℹ️ 命令参数详解
  • --input ./cube.step — 输入文件（刚才生成的 STEP 文件）
  • --export stl — 指定导出格式为 STL
  • --output ./cube.stl — 输出 STL 文件路径
5启动本地 3D 查看器

运行以下命令启动本地 Web 查看器，预览生成的立方体：

npx skills run cad-viewer --file ./cube.step
✅ 预期结果

终端会显示服务启动信息，类似：

Server running at http://localhost:8080

6在浏览器中查看 3D 模型

打开你的浏览器（Chrome、Edge、Firefox 均可），访问以下地址：

http://localhost:8080
💡 浏览器操作提示
  • 鼠标左键拖拽：旋转模型
  • 鼠标右键拖拽：平移模型
  • 鼠标滚轮：缩放模型
📷 需要截图保存！

请按 Print Screen 或 Ctrl+Shift+S（Windows）/Cmd+Shift+4（Mac）截图，保存到 lab01-environment/ 目录，作为环境正常的证明。

7停止查看器（可选）

查看完毕后，回到终端，按 Ctrl+C 停止本地服务器。

Lab 02 验收标准
  • 浏览器打开 http://localhost:8080 能看到立方体
  • cube.step 和 cube.stl 文件存在且大小 > 0

Lab 03：标准件检索

预计用时：15 分钟
🎯 本实验目标

学习使用 step.parts 工具检索标准机械零件（如螺钉、轴承），直接获取标准 STEP 文件。

1创建标准件目录

先进入工作目录并创建存放标准件的文件夹：

cd day1-ai-cad-fundamentals
mkdir -p lab03-std-parts
cd lab03-std-parts
2检索 M6 内六角圆柱头螺钉

运行以下命令，检索 ISO 标准的 M6 内六角圆柱头螺钉：

npx skills run step.parts --query "M6 hex socket head screw" --standard ISO --output ./m6_screw.step
ℹ️ 命令参数详解
  • step.parts — 标准件数据库检索 Skill
  • --query "M6 hex socket head screw" — 检索关键词（英文）
  • --standard ISO — 指定标准体系（ISO/GB/DIN 等）
  • --output ./m6_screw.step — 输出文件路径
3检索 6204 深沟球轴承

继续检索另一个常用标准件 —— 6204 深沟球轴承：

npx skills run step.parts --query "6204 deep groove ball bearing" --standard ISO --output ./bearing_6204.step
4在 FreeCAD 中验证（可选但推荐）

打开 FreeCAD，通过菜单 文件 → 打开 选择刚才下载的 m6_screw.step 或 bearing_6204.step 文件。

💡 FreeCAD 测量操作
  1. 打开 STEP 文件后，在左侧模型树中选中零件
  2. 切换到 Part Workbench
  3. 点击工具栏中的 Measure 按钮
  4. 选择要测量的边/面，验证尺寸是否与标准一致
Lab 03 验收标准
  • 成功输出 .step 文件
  • 在 FreeCAD 中可打开并测量尺寸正确

常见问题排查

API 调用失败？

检查以下项目，按顺序排查：

  1. 检查 .env 文件是否在项目根目录
  2. 检查环境变量是否已导出：
    echo $OPENAI_API_KEY
  3. 检查 API Key 是否有效（是否有余额、是否过期）
  4. 检查网络连接（是否能访问 API 服务端点）

生成的零件尺寸不对？

text-to-cad 默认单位是 mm，但 LLM 可能误解。请遵循以下 prompt 规范：

❌ 不推荐："Create a 20 cube"

✅ 推荐："Create a 20 mm cube"

✅ 更详细："Create a cube with 20 mm side length"

FreeCAD 打不开 STEP？

确保 FreeCAD 版本满足要求：

  • FreeCAD 版本需要 ≥ 1.0
  • 旧版可能不支持某些 AP242 特性
  • 检查方法：FreeCAD 菜单 Help → About FreeCAD

npx 命令执行很慢？

首次运行 npx skills run cad 时，可能需要下载依赖，请耐心等待。可以通过以下方式加速：

  • 确保网络连接稳定
  • 使用国内 npm 镜像（如淘宝镜像）
  • 后续执行会快很多（已缓存）
